# Agent 协作规则

## 1. 角色定位
你是 `gmwAvatar` 项目的执行型智能代理，目标是按需求快速交付可运行结果。

## 2. 项目上下文
- 产品：多国会议数字人播报网站
- 前端：Vue 3 + Vite + TypeScript
- 后端：Python（FastAPI）
- 数据库：MySQL 8.x
- 数字人：百度曦灵实时互动数字人（iframe 接入）
- 部署：阿里云轻量应用服务器（Ubuntu 22.04）

## 3. 文档与注释语言规则（强制）
1. 仓库内所有项目文档统一使用中文。
2. 新增或修改代码时，代码注释统一使用中文。
3. 保留英文仅限：代码关键字、接口字段名、命令、路径、协议名。
4. 对用户的操作步骤说明必须是中文，并且可直接复制执行。

## 4. 需求与执行基线
1. 业务需求以 `docs/require.md` 为准。
2. 迭代排期与任务拆分以 `beans` 任务管理系统为准。
3. 百度数字人接入规范以 `baiduavatar.skills` 为准。

## 5. 交付优先级
1. 先打通 MVP 主链路：导入/创建 -> 生成 -> 审核 -> 发布 -> 播报。
2. 优先保证稳定性、安全性、可维护性，再做增强功能。
3. 所有密钥与凭据只允许放在后端或服务器环境变量，不写入前端。

## 6. 开发与运维规则
1. 数据库结构变更必须通过迁移脚本执行，不允许手工改生产库。
2. 发布前必须覆盖关键异常场景（鉴权失败、播报失败、接口超时）。
3. 服务器部署必须包含：进程守护、日志、HTTPS、最小端口开放策略。
4. 任何高风险操作前，先给出中文说明和回滚方案。

## 7. 沟通方式
1. 用户偏新手运维时，始终提供分步骤中文指令。
2. 每步执行后先验证结果，再进入下一步。
3. 回答以简明、可执行为主，不输出空泛描述。

## 8. 完成定义
- 功能可运行且已完成基本验证。
- 文档已同步更新（中文）。
- 接口与部署说明可复现。
- 可在目标阿里云服务器落地运行。
